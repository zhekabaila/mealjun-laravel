### Critical Patterns

- All models use `HasUuids` trait
- All tables use `uuid('id')->primary()->default(DB::raw('gen_random_uuid()'))`
- Controllers separate public methods (`publicIndex()`, `publicShow()`) from admin methods (`index()`, `store()`)
- Image uploads: always base64 via `CloudinaryService->uploadBase64()`
- WhatsApp: always via `EvolutionApiService->notifyAdmin()` with auto-formatting
- Routes: 3 groups — `/api/auth/*`, `/api/public/*`, `/api/*` (protected)

## Coding Standards to Enforce

### Database & Models

- ✅ UUID primary keys with `gen_random_uuid()` default
- ✅ `use HasUuids` trait on all models
- ✅ `protected $fillable[]` defined
- ✅ `protected $casts[]` for type conversion
- ✅ `protected $hidden[]` for sensitive fields (password_hash)
- ✅ Descriptive relation names (creator, analytics, generatedCaptions)

### Controllers

- ✅ Public methods: `publicIndex()`, `publicShow()`, etc.
- ✅ Admin methods: `index()`, `store()`, `update()`, `destroy()`
- ✅ Custom methods: `toggleFeatured()`, `markAsRead()`, `reorder()`
- ✅ Service injection via constructor
- ✅ Consistent response format (json with 200/201/204 status)

### Validations

- ✅ Custom messages in Indonesian
- ✅ Base64 validation: `regex:/^data:image\/(jpeg|png|webp|gif);base64,/`
- ✅ UUID validation: `uuid|exists:table,column`
- ✅ Enum validation: `in:value1,value2,value3`

### Images

- ✅ Only via base64: `data:image/type;base64,...`
- ✅ Folders: mealjun/products, mealjun/avatars, mealjun/gallery, mealjun/about
- ✅ Return: `['url' => string, 'public_id' => string]`

### WhatsApp

- ✅ Auto-format: `08xx` → `628xx`
- ✅ Always validate number before sending
- ✅ Use `notifyAdmin()` method (includes validation)
- ✅ Never block main process on failure (try-catch, log)
- ✅ Format with emoji and bold text (`*text*`)

## Common Tasks & Approach

### Task: Add New Entity (e.g., Menu Table)

**Steps**:

1. Create migration: `php artisan make:migration create_menus_table`
2. Define schema with UUID primary key, timestamps, proper indexes
3. Create model: `php artisan make:model Menu` with `HasUuids`, `$fillable`, `$casts`
4. Create controller: `php artisan make:controller Api/MenuController`
    - Separate `publicIndex()` from `index()`
    - Add CRUD methods
    - Inject services if image uploads needed
5. Add routes in `routes/api.php`:
    - Public: `GET /api/public/menus`
    - Admin: `GET /api/menus`, `POST /api/menus`, etc.
6. Create seeder if needed
7. Test with cURL or Postman

### Task: Integrate WhatsApp Notification for New Event

**Steps**:

1. Inject `EvolutionApiService $evolutionApi` in controller
2. Get admin phone from `AboutInfo::first()->whatsapp_number`
3. Format message with emoji and bold text
4. Call: `$this->evolutionApi->notifyAdmin($phone, $message)`
5. Wrap in try-catch, don't block main response

### Task: Handle Image Upload

**Steps**:

1. Validate: `'image_base64' => 'required|string|regex:/^data:image\/(jpeg|png|webp|gif);base64,/'`
2. Inject `CloudinaryService $cloudinary` in controller
3. Call: `$upload = $this->cloudinary->uploadBase64($base64, 'mealjun/folder')`
4. Use `$upload['url']` and `$upload['public_id']` in response
5. On update: check if new base64 provided, upload if yes

## Error Handling Standards

### Database Errors

- Use `findOrFail()` for 404 handling
- Let validation exceptions bubble up (Laravel handles automatically)
- Return 500 for unexpected errors (log them)

### Service Errors

- Cloudinary: throw Exception if upload fails
- WhatsApp: catch Exception, log, return false (don't block)

### Response Format

```php
// Success (200 implicit)
return response()->json($data);

// Created (201)
return response()->json($data, 201);

// No Content (204)
return response()->json([], 204);

// Let Laravel handle validation errors (422)
$request->validate([...]);  // Throws ValidationException
```

## Known Constraints & Decisions

| Constraint      | Decision                                      |
| --------------- | --------------------------------------------- |
| Image Storage   | Cloudinary only (base64 upload, no multipart) |
| Primary Keys    | UUID (never auto-increment)                   |
| Database        | PostgreSQL required (uuid-ossp extension)     |
| Authentication  | Sanctum tokens (no session-based auth)        |
| WhatsApp API    | Evolution API only (checked + validated)      |
| API Versioning  | Not versioned (`/api/` is final structure)    |
| Soft Deletes    | Not used (hard delete only)                   |
| Admin Audit Log | Not implemented (out of scope)                |
| CORS            | Not configured in detail (to be clarified)    |
